Fortune Oracle

一个基于传统隐喻体系的结构化解读系统
不做预言，不做裁决
只提供可解释的观察视角

项目愿景

Fortune Oracle 是一个将
传统命理隐喻体系 与 现代工程化 AI 架构 相结合的解释型系统。

本项目关注的不是“命中注定的结果”，
而是如何将长期存在的传统体系，
拆解为可计算、可引用、可审计的结构化模型，
并通过 AI 将这些结构转译为可读的解释文本。

Fortune 在这里不是“未来事件”，
而是结构、关系与倾向的组合。


Fortune Oracle 是什么 / 不是什么
✅ Fortune Oracle 是什么

一个 解释型系统，而非决策系统

一个基于 显式规则与来源 的结构化解读工具

一个强调 可解释性、一致性与边界控制 的 AI 架构实践

一个只描述 倾向、模式与结构关系 的系统

一个工程优先、可验证、可演进的实现
核心设计原则
1️⃣ 描述，而非断言

系统只输出结构性描述与可能倾向，不输出必然结论。

2️⃣ 客观语言

避免情绪引导、权威压迫或宿命化表达。

3️⃣ 可解释性优先

所有解读必须可以回溯到：

明确的结构化输入

明确的规则定义

明确的来源标识

4️⃣ 决策权始终属于用户

系统不提供“应该如何选择”的指令，只提供观察视角。

5️⃣ 工程纪律

严格区分以下职责边界：

确定性计算

规则与来源管理

语言生成与表达

系统整体架构

Fortune Oracle 不是一个单 Prompt 聊天机器人，
而是一个分层、可控、可验证的系统。

多 Agent / 模块分工
模块 / Agent	职责
输入采集模块	输入校验、规范化
计算引擎	生成确定性、结构化结果
规则管理模块	维护规则定义与来源
解释 Agent	将结构化结果转为自然语言
校验 Agent	防止绝对化、越界与幻觉
可解释性与可追溯性

系统中的每一条解读，都应当具备：

对应的结构化事实

明确的规则标识（rule_id）

明确的概念或文献来源

当条件不足或规则冲突时，系统应明确说明：

当前条件不足以支持明确判断。

非预言与责任声明

Fortune Oracle 不从事任何形式的占卜或预言。

系统不会：

预测具体未来事件

判断人生“好坏”

宣布不可改变的结论

Fortune Oracle 提供的是：

结构化观察

符号化解释

可供参考的分析视角

请将系统输出视为分析结果，而非结论。
